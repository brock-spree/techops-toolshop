Spreelet Architecture

Tech Ops Toolshop — layers, surfaces, components, lifecycle, entities, and build enforcement

1. Design System Layer Model
Tech Ops Toolshop
Domain-scoped container — Registry reader, Spreelet Card grid, Spark Launcher, auth context
Parameterized by TOOLSHOP_DOMAIN. One codebase, multiple domain instances. Reads spreelet.json manifests.
↓
Plugin Skills
Build-time enforcement — spreelet-patterns (creation guide), spreelet-readiness (audit + fix), entity-domain (modeling)
Lives in tech-ops plugin. Used by App Builder + Claude Code. Standards flow through the build process.
↓
Spreelet Kit
Shared TechOps patterns — StatusCapsule, FreshnessIndicator, ConfidenceBadge, TrendSparkline, EntityPill, EditableCell, SkeletonLoader, ErrorBanner, EmptyState
Domain-specific color tokens (DQ, freshness, confidence, trends, entities). Layout conventions. Interaction patterns.
↓
Spreeform
Design system foundation — Atoms (Button, Input, Card, Dialog, Table...) + Blocks (Sidebar, Page, FlexTable, ThemeToggle)
Built on Radix UI + Tailwind CSS. ~60 atom components, 6 block components. Themed via CSS variables.
↓
App Framework
TanStack ecosystem — TanStack Start (SSR + Server Functions) + Router (file-based) + React Query (data fetching)
React 19, TypeScript, Vite, Nitro backend, Bun runtime, Zod validation.
↓
Data Platform
Trino / Starburst Galaxy (read-only analytics) + PostgreSQL (per-app state, entities)
Server functions to Trino for warehouse queries. Postgres for entities, config, telemetry. Standardized entity schema.
2. The Three Surfaces — Card → Spark → Full App
Spreelet Card

Auto-Generated (No Code)

Browsing state in the Toolshop grid. Rendered entirely from spreelet.json manifest metadata.

  • Name + icon + description + tags
  • Optional live summary endpoint
  • "Open" + "Launch Spark" CTAs
  • Zero Spreelet code required
→
Spreelet Spark

Plug-and-Play Component

Focused use in modal, panel, or embedded context. Built by author using Spreelet Kit templates. Web Component for portability.

  • Core functionality only
  • No navigation chrome
  • "Go to full app" deep-link
  • Embeddable on intranet / other apps
→
Full App

Standalone Experience

Complete app at its own URL. All features, full navigation. Works independently of the Toolshop.

  • All features and admin
  • Full navigation (Spreeform Sidebar)
  • Deep-link from Spark state
  • Independently deployable on Kova

4. Spreelet Lifecycle — Create → Surface → Enhance → Converge

Create

Minutes

Build with App Builder or Claude Code + tech-ops plugin. spreelet-patterns skill guides creation.

Output: Working app on Kova
→

Surface

5 minutes

Add spreelet.json manifest. spreelet-readiness skill generates it. Card appears in Toolshop automatically.

Output: Visible in Toolshop
→

Enhance

Hours to days

Build Spark component. Adopt Spreelet Kit. Declare entity domains. Run spreelet-readiness audit.

Output: Card + Spark + Entities
→

Converge

Ongoing

Overlapping tools merge organically. Best patterns adopted. Usage data guides decisions.

Output: Refined ecosystem
